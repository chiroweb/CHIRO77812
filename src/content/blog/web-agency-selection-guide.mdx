## 웹 에이전시 선택에서 가장 흔한 실수

홈페이지 제작 업체를 고르는 대부분의 사람들은 포트폴리오만 봅니다. 디자인이 예쁘고, 가격이 적당하고, 납기가 빠르면 계약합니다. 이 방식으로 선택했다가 납품 후 후회하는 경우가 많습니다.

겉으로 보이는 것과 실제 구조적 품질은 다릅니다. 디자인은 멋지지만 구글에서 검색이 안 되거나, 모바일에서 레이아웃이 깨지거나, 코드 소유권이 없어서 에이전시를 떠나지 못하는 상황이 생깁니다.

계약 전에 확인해야 할 7가지를 기준으로 정리합니다.

---

## 기준 1: 포트폴리오의 실제 성능을 확인하라

포트폴리오를 볼 때 디자인만 보면 안 됩니다. 제시된 포트폴리오 사이트를 구글 PageSpeed Insights에 직접 입력해보면 됩니다.

90점 이상이면 코드 품질이 높은 업체입니다. 70점 이하면 빌더 기반이거나 최적화 작업을 하지 않는 업체일 가능성이 높습니다.

모바일 점수도 확인하세요. PC 점수는 높은데 모바일 점수가 낮은 경우가 많습니다. 2026년 현재 전체 웹 트래픽의 65%가 모바일에서 발생합니다. 구글도 모바일 우선 색인(Mobile-First Indexing)을 적용합니다.

---

## 기준 2: 코드 소유권 조항을 계약서에서 확인하라

제작 완료 후 코드의 소유권이 누구에게 있는지를 계약서에서 확인해야 합니다.

일부 에이전시는 코드 소유권을 자신들이 보유하는 조항을 계약서에 포함합니다. 이 경우 다른 업체로 유지보수를 이전하거나, 직접 수정하거나, 다른 서버에 올리는 것이 어렵습니다. 에이전시와의 관계가 틀어져도 선택지가 없어집니다.

납품물은 소스코드 전체를 포함해야 하고, 해당 코드를 고객이 자유롭게 사용·수정·이전할 수 있어야 합니다. 이 내용이 계약서에 명시되어 있지 않으면 계약 전에 요청해야 합니다.

---

## 기준 3: SEO와 AEO가 포함된 작업인지 물어보라

"SEO 최적화 포함"이라는 문구는 거의 모든 에이전시의 서비스 소개에 있습니다. 하지만 실제로 무엇이 포함되는지는 에이전시마다 다릅니다.

최소한 다음을 확인해야 합니다.

- 구조화 데이터(JSON-LD)를 페이지별로 적용하는가.
- llms.txt를 기본 포함하는가.
- 메타 태그(title, description, OG)를 페이지별로 설정하는가.
- 이미지 alt 태그를 작성하는가.
- 시맨틱 HTML(header, main, article, section)을 사용하는가.

AEO(AI 검색 최적화)를 언급하지 못하는 업체라면, AI 검색 트래픽에 대응할 준비가 안 된 업체입니다. 2026년 기준으로 ChatGPT와 Perplexity 같은 AI 검색은 선택이 아닌 필수 채널입니다.

---

## 기준 4: 유지보수 계약 구조를 파악하라

홈페이지는 납품 후가 더 중요합니다. 텍스트 수정, 이미지 교체, 새 페이지 추가, 오류 수정 등 지속적인 관리가 필요합니다.

유지보수 계약에서 확인해야 할 사항은 다음과 같습니다.

- 월 유지보수 비용이 얼마인가. 그 비용에 무엇이 포함되는가.
- 수정 요청 처리 시간은 얼마나 걸리는가.
- 비상 상황(사이트 다운, 오류)에 대응하는 시간은 얼마인가.
- 계약 해지 시 코드와 데이터를 어떻게 인계받는가.

유지보수를 과도하게 묶어두는 구조도 문제입니다. 월 30~50만원씩 받으면서 실제로 하는 일이 텍스트 수정 몇 건인 경우가 있습니다. CMS를 연결하면 기본적인 수정은 직접 할 수 있습니다.

---

## 기준 5: 반응형 구현 방식을 확인하라

반응형 디자인은 2026년 기준으로 기본 중의 기본입니다. 중요한 것은 "모바일에서 보인다"가 아니라 "모바일에서 최적화된다"는 점입니다.

확인 방법은 간단합니다. 포트폴리오 사이트를 실제 스마트폰으로 열어보면 됩니다. 텍스트가 너무 작거나, 버튼을 누르기 어렵거나, 가로 스크롤이 생기면 반응형이 제대로 구현되지 않은 것입니다.

Google Mobile-Friendly Test에서 포트폴리오 URL을 검색해보는 것도 방법입니다.

---

## 기준 6: 호스팅 구조와 비용을 확인하라

사이트가 어디에 올라가는지도 중요합니다. 에이전시가 자체 서버를 사용하는 경우, 관계가 끊어지면 사이트에 문제가 생길 수 있습니다.

고성능 호스팅(Vercel, AWS, Cloudflare Pages)을 사용하는지, 공유 호스팅을 사용하는지에 따라 속도와 안정성이 달라집니다. 공유 호스팅은 여러 사이트가 하나의 서버를 나눠 쓰는 구조라 트래픽이 몰리면 느려집니다.

호스팅 계정이 고객 명의로 되어 있는지도 확인해야 합니다. 에이전시 계정에 등록된 경우, 계약이 끊기면 호스팅이 종료됩니다.

---

## 기준 7: AEO 대응 능력을 직접 물어보라

"AI 검색 최적화를 어떻게 합니까?"라고 물어보면 됩니다.

답변에서 llms.txt, 구조화 데이터, robots.txt AI 로봇 설정, 시맨틱 HTML, Core Web Vitals가 언급되지 않으면 AEO에 대한 준비가 없는 업체입니다.

2026년 현재 검색 트래픽의 구조가 바뀌고 있습니다. 구글 검색 결과에 AI Overview가 삽입되고, ChatGPT와 Perplexity가 검색 채널로 자리잡았습니다. 이 채널에 대응하지 못하는 사이트는 새로운 트래픽 흐름에서 보이지 않게 됩니다.

## 치로의 기준

치로는 위 7가지 기준 모두에 명확하게 답합니다.

코드 소유권은 납품 시 고객에게 완전 귀속됩니다. SEO와 AEO 설정은 기본 포함입니다. 호스팅은 Vercel 기반으로 고객 계정에 설정합니다. 모든 사이트는 Lighthouse 모바일 점수 90점 이상을 목표로 제작합니다.

[치로 포트폴리오](/portfolio)에서 실제 구현 사례를 확인할 수 있습니다. 사이트 구조와 검색 성능이 어떻게 설계되었는지를 직접 검토해보세요.

계약 전에 확신이 서지 않는다면, [무료 진단](/free-diagnosis)을 통해 현재 사이트의 구조적 문제를 먼저 파악하는 것도 방법입니다.

## 자주 묻는 질문

**포트폴리오가 없는 신생 에이전시는 피해야 하나요?**
포트폴리오 수가 중요한 게 아닙니다. 작은 규모라도 코드 구조, 검색 최적화 방식, 코드 소유권 정책이 명확하다면 신뢰할 수 있습니다. 포트폴리오가 많아도 빌더 기반이거나 SEO 기준이 없는 업체보다 낫습니다.

**가격이 너무 낮은 업체는 무조건 피해야 하나요?**
가격 자체보다 무엇이 포함되는지가 더 중요합니다. 50만원에 커스텀 코딩을 한다는 것은 불가능에 가깝습니다. SEO, AEO, 반응형, 코드 소유권 등이 포함되지 않거나 빌더 기반일 가능성이 높습니다.

**에이전시와 프리랜서 중 어디에 맡겨야 하나요?**
규모보다 역량이 중요합니다. 에이전시여도 수준이 낮을 수 있고, 1인 프리랜서여도 높은 퀄리티를 제공할 수 있습니다. 위의 7가지 기준을 동일하게 적용해서 판단하면 됩니다.

**계약서 없이 구두로 진행해도 되나요?**
안 됩니다. 코드 소유권, 납기일, 수정 횟수, 유지보수 조건은 반드시 서면으로 남겨야 합니다. 분쟁이 생겼을 때 구두 합의는 증거가 되지 않습니다.

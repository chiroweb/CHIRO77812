## "Crawled - currently not indexed"가 떴다면

구글 서치콘솔에 사이트맵을 제출했는데 색인이 안 됩니다. 가장 흔한 오해는 "기다리면 된다"입니다. 1년이 지나도 안 됩니다. 원인이 따로 있습니다.

실무에서 만나는 색인 실패 원인 5가지를 정리합니다. 검색해도 잘 안 나오는 진짜 패턴입니다.

## 원인 1. 콘텐츠가 "충분히 다르지 않다"

구글은 2024년부터 thin content 판정을 더 엄격하게 합니다. 같은 업종 사이트들이 비슷한 구조와 비슷한 텍스트(설립연도, 비전, 서비스 4종)를 가지면 둘 중 하나만 색인합니다.

해결: 페이지마다 다른 업체에는 없는 정보 1개를 넣습니다. 가격 범위, 응답 시간, 구체 사례 같은 1차 데이터.

## 원인 2. 내부 링크가 0개인 고아 페이지

사이트맵에 들어가 있어도, 다른 페이지에서 그 페이지로 가는 링크가 0개면 구글은 "중요하지 않다"고 판단합니다. 이를 고아 페이지(orphan page)라고 합니다.

해결: 새 페이지를 만들면 최소 3개 페이지에서 그 페이지로 내부 링크를 답니다. 메인 페이지에서 1개, 카테고리 페이지에서 1개, 관련 블로그 글에서 1개가 기본 패턴입니다.

## 원인 3. 빌더 자동 생성 noindex 태그

아임웹·카페24의 일부 페이지(주문 완료, 검색 결과, 태그 페이지)는 자동으로 `<meta name="robots" content="noindex">`가 붙습니다. 사용자는 모릅니다.

해결: 페이지 우클릭 → 페이지 소스 보기 → "noindex" 검색. 발견되면 빌더 설정에서 해제하거나, 빌더가 허용 안 하면 그 페이지는 색인 불가능입니다. [빌더의 SEO 한계 글](/blog/imweb-cafe24-seo-limits)에서 더 다룹니다.

## 원인 4. JavaScript 렌더링 지연

리액트·뷰로 만든 SPA(싱글 페이지 앱)는 초기 HTML이 비어 있고 자바스크립트가 콘텐츠를 그립니다. 구글봇은 자바스크립트를 실행하지만, 두 번째 패스로 실행하므로 색인이 늦거나 누락됩니다.

해결: SSR(서버 사이드 렌더링) 또는 SSG(정적 생성)로 전환합니다. Next.js·Nuxt가 이 문제를 해결하는 기본 솔루션입니다.

## 원인 5. 모바일 콘텐츠가 PC와 다르다

구글은 모바일 우선 색인(Mobile-First Indexing)을 사용합니다. PC에는 보이지만 모바일에서 "더보기" 뒤에 숨어 있는 콘텐츠는 색인 가중치가 낮습니다.

해결: 모바일에서 hidden·display:none·collapse로 숨긴 콘텐츠를 점검합니다. PC와 동일한 텍스트가 보이도록 구조를 바꿉니다.

## 색인 강제 트리거가 필요할 때

5가지를 다 점검했는데도 안 되면 [발행 직후 색인 강제 루틴](/blog/indexing-trigger-5min-routine) 글을 참고합니다. URL Inspection의 숨은 사용 순서가 따로 있습니다.

## 자주 묻는 질문

**Sitemap.xml은 왜 효과가 약한가요?**
사이트맵은 "이 URL이 있다"를 알려주는 신호일 뿐이고, 색인 여부는 구글이 페이지의 가치를 별도로 판단합니다. 사이트맵보다 내부 링크와 콘텐츠 차별성이 더 강한 신호입니다.

**색인된 페이지가 갑자기 사라지는 경우는?**
중복 콘텐츠 판정, 코어 업데이트, 또는 페이지 품질 하락이 원인입니다. 마지막 수정일이 1년 이상 지난 페이지가 우선 탈락 대상입니다.

이커머스 사이트 평균 전환율은 **2~3%**. 서비스업은 1~4%. 즉 광고로 100명을 데려와도 97~99명은 사지 않고 떠납니다. 이 97~99명을 다시 불러오는 것이 **리타겟팅(Retargeting)** 광고입니다.

리타겟팅은 광고 효율의 핵심입니다. 일반 광고 대비 CVR이 **2~7배** 높고, CPA가 **절반**입니다. 그런데 이 강력한 무기를 쓰려면 조건이 있습니다. **자사 홈페이지에 픽셀(Pixel)이 설치**되어 있어야 합니다.

---

## 픽셀이 무엇인가

픽셀은 홈페이지에 심는 **추적 코드**입니다. 방문자의 행동을 광고 플랫폼(메타·구글·네이버)에 전달합니다.

### 주요 픽셀

| 광고 플랫폼 | 픽셀 이름 | 추적 데이터 |
|------------|----------|------------|
| Meta (페이스북·인스타) | Meta Pixel | 페이지 뷰, 장바구니, 구매, 검색 |
| Google Ads | Google Tag (Ads Conversion) | 전환, 재방문자 분류 |
| Google Analytics | GA4 Tag | 세션, 이벤트, 퍼널 |
| Naver | Naver Premium Log | 유입 경로, 전환 |
| TikTok | TikTok Pixel | 행동, 구매 |
| Kakao | Kakao Pixel | 행동, 전환 |

픽셀이 설치된 홈페이지를 방문하면, 방문자의 브라우저에 **쿠키**가 저장됩니다. 이 쿠키로 광고 플랫폼은 "이 사람이 내 사이트에 왔었다"는 사실을 기억합니다.

---

## 리타겟팅 광고의 작동 구조

### 1. 방문 → 쿠키 저장

```
사용자가 내 홈페이지 방문
  ↓
메타 픽셀이 방문 이벤트 기록
  ↓
Facebook 서버에 "이 사용자 ID가 내 사이트 방문" 저장
```

### 2. 광고 노출

```
사용자가 Facebook·Instagram 접속
  ↓
Facebook이 "이 사용자는 A 브랜드 사이트 방문 이력 있음" 확인
  ↓
A 브랜드의 리타겟팅 광고가 사용자 피드에 노출
```

### 3. 재방문 → 전환

```
사용자가 광고 클릭 → A 브랜드 사이트 재방문
  ↓
첫 방문 때 봤던 제품에 대한 관심 잔존
  ↓
평균 CVR 7~15% (신규 방문 2~3%의 3~5배)
```

## 홈페이지가 없으면 어떻게 되나

### 시나리오: 쿠팡 스토어만 운영

당신이 광고를 돌려 쿠팡 내 스토어로 사용자를 보냅니다. 결과:

- **메타 픽셀 설치 불가** — 쿠팡 페이지에 내 코드 못 넣음
- **방문자 데이터 수집 불가** — 쿠팡이 모두 가져감
- **리타겟팅 광고 불가** — 누가 왔는지 모름
- **"유사 타겟(Lookalike)" 광고도 불가** — 기반 데이터 없음

광고로 쿠팡 매출은 올렸지만, **당신에게 남은 자산은 0**. 다음 달에도 같은 광고를 처음부터 돌려야 합니다.

### 시나리오: 자사 홈페이지 운영

광고로 자사몰에 유입. 결과:

- 픽셀이 방문자 10,000명 데이터 수집
- 장바구니에 넣고 나간 1,200명 식별
- 상품 상세 본 5,800명 식별
- **리타겟팅 광고로 이 모집단에 정밀 타겟팅 가능**

한 번의 광고비로 **다음 광고까지 효율이 2~5배** 올라갑니다.

## 리타겟팅 실제 수치 — 일반 광고 vs 리타겟팅

치로가 운영한 이커머스 사이트 3곳 평균 (2026년 1분기).

| 지표 | 프로스펙팅(신규) 광고 | 리타겟팅 광고 |
|------|--------------------|--------------|
| CTR (클릭률) | 1.2% | 3.8% |
| CVR (전환율) | 1.8% | 9.2% |
| CPC (클릭당 비용) | 1,200원 | 850원 |
| CPA (전환당 비용) | 66,700원 | 9,240원 |
| ROAS | 280% | 820% |

**리타겟팅이 CPA를 1/7로 낮추고 ROAS를 3배로 올립니다.**

## 픽셀 설치의 4가지 수준

홈페이지가 있어도 픽셀 설치 수준에 따라 효과가 극명하게 다릅니다.

### 수준 1: 페이지 뷰만 추적

```html
<script>
  fbq('track', 'PageView');
</script>
```

방문자가 "왔다"는 사실만 기록. 가장 기본.

### 수준 2: 주요 이벤트 추적

```javascript
fbq('track', 'ViewContent', { content_ids: ['123'] });
fbq('track', 'AddToCart', { value: 35000, currency: 'KRW' });
fbq('track', 'InitiateCheckout');
fbq('track', 'Purchase', { value: 89000, currency: 'KRW' });
```

장바구니·결제 시작·구매까지 세분화. 퍼널 이탈 지점 파악 가능.

### 수준 3: Conversion API (서버 사이드)

iOS 14.5 이후 쿠키 차단 강화로 브라우저 픽셀 정확도 하락. 해결책은 **서버에서 직접 이벤트 전송**하는 Conversion API. 광고 최적화 정확도 40~60% 복원.

### 수준 4: Enhanced Conversions + 오프라인 전환

이메일·전화번호를 해시 처리해 광고 플랫폼에 전송. 오프라인 구매까지 온라인 광고에 연결. 성숙 단계 광고주 필수.

**빌더(아임웹·카페24)는 수준 1~2까지만 가능**합니다. 수준 3~4는 코드 직접 제어 필요.

## 픽셀 설치의 실전 체크리스트

치로가 모든 프로젝트에서 런칭 전 확인하는 10가지.

- [ ] Meta Pixel ID 발급 및 사이트 전 페이지 설치
- [ ] Google Analytics 4 설치 및 전자상거래 이벤트 연동
- [ ] Google Ads 전환 태그 설치
- [ ] Naver Premium Log (네이버 광고 시)
- [ ] PageView, ViewContent, AddToCart, Purchase 이벤트 발송 확인
- [ ] Facebook Events Manager에서 실시간 이벤트 수신 확인
- [ ] GA4 DebugView에서 이벤트 검증
- [ ] Conversion API 서버 이벤트 전송 테스트
- [ ] iOS Safari에서 픽셀 작동 확인 (쿠키 제한 환경)
- [ ] 개인정보 처리방침·쿠키 배너 적용 (법적 요구)

---

## 빌더 사이트에서 픽셀을 설치할 때의 한계

- **수준 1만 대부분 가능** (페이지 뷰)
- 장바구니·결제 이벤트는 일부 빌더만 부분 지원
- Conversion API 서버 사이드 구현 **불가**
- 쇼핑몰 전용 이벤트(ProductDetail, AddPaymentInfo 등) 커스터마이징 **불가**
- 이벤트 데이터 품질 평가(Event Match Quality) 40점대 (권장 80+)

결과: 광고 플랫폼이 학습할 데이터가 부족해 최적화가 작동하지 않습니다. 결국 광고 효율이 **평균 30~50% 낮은** 상태로 고착.

## 자주 묻는 질문

### Q. 픽셀 설치만 하면 바로 리타겟팅이 작동하나요?

모집단이 **최소 1,000명** 쌓여야 의미 있는 리타겟팅이 가능합니다. 월 방문자 3,000명 기준 2~4주 소요. Meta 광고는 기본 1,000명 모집단 요구.

### Q. 픽셀 설치가 GDPR·개인정보보호법 위반 아닌가요?

쿠키 동의(Cookie Consent) 배너를 설치하고 사용자 동의를 받으면 문제없습니다. 한국은 2024년부터 전자상거래법 시행령에 **쿠키 배너 의무화**됐습니다. 치로는 모든 프로젝트에 기본 적용.

### Q. 여러 픽셀(Meta, Google, Naver)을 동시에 설치하면 사이트가 느려지지 않나요?

비동기 로딩(async) + 서버 사이드 태그 관리(Google Tag Manager Server-Side)를 쓰면 성능 영향 미미. 치로 기준 픽셀 5개 동시 운영 시 LCP 증가 **0.1~0.2초** 수준.

## 결론

리타겟팅은 광고비 효율을 결정짓는 가장 강력한 레버입니다. 그리고 이 레버를 당기려면 **자사 홈페이지 + 제대로 된 픽셀 설치**가 전제입니다. 지금 광고비가 새고 있다면, 픽셀 설치 수준부터 확인하세요.

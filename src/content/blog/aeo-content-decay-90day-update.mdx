## AEO 콘텐츠는 발행 후 90일부터 인용이 떨어진다

SEO 콘텐츠는 한 번 상위에 오르면 6~24개월 트래픽이 유지됩니다. AEO는 다릅니다. 치로웹디자인의 자체 추적 결과 AEO 콘텐츠의 평균 인용 수명은 90~120일입니다. 그 이후로 인용률이 60~80%로 감소하고, 6개월 시점에는 30~40% 수준이 됩니다.

이유는 AI 검색이 신선도를 우선 신호로 사용하기 때문입니다. 같은 주제로 더 최신 콘텐츠가 발행되면 AI는 그쪽으로 출처를 교체합니다. AEO 콘텐츠는 발행이 아닌 유지보수가 트래픽을 결정합니다.

---

## 인용률 감소 패턴

치로웹디자인이 16개 AEO 글을 12개월 추적한 데이터입니다.

| 발행 후 경과 | 평균 인용률 | 전월 대비 변화 |
|---------|----------|------------|
| 1개월 | 24% | - |
| 3개월 | 38% (정점) | +14%p |
| 6개월 | 22% | -16%p |
| 9개월 | 14% | -8%p |
| 12개월 | 9% | -5%p |

발행 후 3개월에 정점을 찍고 그 이후로 빠르게 감소합니다. 정점 시점에 업데이트하면 추가 90일 수명이 연장됩니다. 정점을 놓치면 회복이 어렵습니다.

## 업데이트가 효과적인 시점

업데이트는 발행 후 80~100일 사이가 가장 효과적입니다. 이 시점에 콘텐츠를 갱신하면 AI 크롤러가 modifiedDate 신호를 인식하고 재인덱싱합니다. 인용률이 발행 직후 수준으로 회복됩니다.

너무 일찍(30일 이내) 업데이트하면 효과가 작습니다. AI 크롤러가 첫 인덱싱을 완료하기 전이라 변경 감지가 약합니다. 너무 늦게(180일 이후) 업데이트하면 이미 다른 콘텐츠가 자리를 차지해 회복이 어렵습니다.

---

## 효과적인 업데이트 작업 6가지

단순히 "수정일만 변경"하는 것은 효과가 없습니다. AI는 실제 콘텐츠 변화를 감지합니다. 다음 6가지 작업 중 2~3가지를 동시에 적용합니다.

1. 통계 숫자를 최신 데이터로 교체. 2024년 자료를 2026년 자료로.

2. 새로운 도구·서비스 항목 추가. 산업 변화 반영.

3. FAQ에 2~3개 질문 추가. 신규 사용자 질문 패턴 반영.

4. H2 1개 추가. 본문에 신규 섹션 1개 삽입.

5. 표 1개 보강. 비교 항목 추가 또는 수치 갱신.

6. 외부 참조 링크 1~2개 추가. 최신 권위 소스 인용.

치로웹디자인은 [AEO·SEO 통합 설계 서비스](/services/seo-aeo)에서 분기별 콘텐츠 업데이트 작업을 표준에 포함합니다.

## 업데이트 vs 신규 발행 — 결정 기준

오래된 글을 업데이트할지, 새 글을 발행할지 결정하는 기준입니다.

| 조건 | 업데이트 | 신규 발행 |
|------|--------|--------|
| 핵심 주제가 동일 | 업데이트 | - |
| 주제가 확장됨 | - | 신규 |
| 글 길이가 1,500자 미만 | 업데이트 (보강) | - |
| 글 구조가 청킹에 불리 | 업데이트 (재구성) | - |
| URL이 SEO 트래픽을 받고 있음 | 업데이트 (URL 유지) | - |
| 새 키워드 군집 진입 | - | 신규 |

URL이 트래픽을 받고 있는 글은 절대 삭제하거나 URL 변경을 하지 않습니다. 같은 URL에서 콘텐츠만 갱신하면 누적 권위가 보존됩니다.

## 자동 알림 시스템 구축

수동으로 발행일을 추적하면 업데이트 시점을 놓치기 쉽습니다. 자동 알림이 효율적입니다.

frontmatter의 publishedAt을 기준으로 발행 후 80일이 지난 글에 알림을 보내는 스크립트입니다.

```typescript
import { getAllMdxPosts } from "@/lib/mdx";

const posts = await getAllMdxPosts();
const now = Date.now();

const needsUpdate = posts.filter((p) => {
  const published = new Date(p.frontmatter.publishedAt).getTime();
  const daysSince = (now - published) / (1000 * 60 * 60 * 24);
  return daysSince >= 80 && daysSince <= 100 
         && !p.frontmatter.updatedAt;
});

console.log(`업데이트 필요 글 ${needsUpdate.length}개`);
```

GitHub Actions 또는 Vercel Cron으로 매주 실행하면 자동 알림이 됩니다.

## 부패 측정 지표

콘텐츠 부패를 정량적으로 측정하는 3가지 지표입니다.

첫째, 인용률 변화. [AEO 성과 측정](/blog/aeo-kpi-citation-rate)의 인용률을 콘텐츠별로 분리해 추적합니다.

둘째, AI 트래픽 변화. GA4에서 페이지별 AI referrer 세션을 월간 비교합니다.

셋째, 답변 출처 위치. ChatGPT 답변에서 출처 순서 중 자사 사이트의 위치(1번째 vs 5번째)를 추적합니다. 순위가 떨어지면 부패 신호입니다.

## 자주 묻는 질문

**모든 글을 90일마다 업데이트해야 하나요?**
트래픽 상위 20% 글만 90일 주기로 업데이트합니다. 하위 80% 글은 6개월 주기로 충분합니다. 업데이트 가치가 없는 글은 archive 처리하거나 다른 글로 통합합니다.

**SEO도 같은 주기로 업데이트가 필요한가요?**
SEO는 6~12개월 주기가 일반적입니다. AEO는 그보다 짧은 90일 주기가 필요합니다. 같은 글에 두 가지 작업을 동시 적용하는 것이 효율적입니다. SEO 키워드 보강과 AEO 표현 다양화를 동시에 합니다.

**업데이트 시 URL을 바꾸면 안 되나요?**
URL 변경은 누적 권위를 모두 잃는 작업입니다. 정말 필요하면 301 리다이렉트로 처리합니다. 가능하면 URL 유지가 원칙입니다. 콘텐츠만 갱신하고 URL은 그대로 둡니다.

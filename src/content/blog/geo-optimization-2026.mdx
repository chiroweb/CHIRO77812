## 검색 최적화의 세 번째 층이 등장했다

2024년까지 검색 최적화는 한 가지였습니다. 구글에서 상위 노출되는 것. 이걸 SEO라고 불렀습니다.

2025년, AI 검색 서비스가 폭발적으로 성장하면서 두 번째 층이 추가되었습니다. ChatGPT나 Perplexity가 답변할 때 내 사이트를 출처로 인용하게 만드는 것. 이걸 AEO라고 부릅니다.

2026년, 세 번째 층이 등장했습니다. **GEO**(생성형 검색 최적화)입니다.

---

## GEO는 AEO와 어떻게 다른가

AEO는 "AI가 내 사이트를 인용하게 만드는 것"입니다. GEO는 한 걸음 더 나갑니다. **AI가 생성하는 답변의 내용 자체에 영향을 주는 것**입니다.

구체적으로 비교하면:

| 구분 | SEO | AEO | GEO |
|------|-----|-----|-----|
| 목표 | 검색 결과 순위 | AI 답변 출처 인용 | AI 답변 내용 형성 |
| 대상 | 구글, 네이버 | ChatGPT, Perplexity | 구글 AI 오버뷰, 빙 코파일럿 |
| 핵심 기술 | 메타 태그, 백링크 | 구조화 데이터, llms.txt | 권위 시그널, 인용 네트워크 |
| 결과 | 파란 링크 노출 | 출처로 링크 표시 | AI 답변 자체에 정보 반영 |

GEO가 중요해진 이유는 구글의 변화 때문입니다. 구글은 2025년부터 검색 결과 최상단에 **AI 오버뷰**를 표시하기 시작했습니다. 사용자가 검색하면 파란 링크 목록이 아니라, AI가 생성한 답변이 먼저 나옵니다.

이 AI 답변에 당신의 정보가 포함되느냐 아니냐가 GEO의 핵심입니다.

---

## 구글 AI 오버뷰가 바꾼 것

구글에서 "홈페이지 제작 비용"을 검색하면, 예전에는 10개의 파란 링크가 나왔습니다.

2026년에는 다릅니다. 최상단에 AI가 생성한 요약 답변이 먼저 나옵니다. "홈페이지 제작 비용은 규모에 따라 50만 원~500만 원 수준이며..." 같은 답변입니다.

이 답변 아래에 출처 사이트 3~5개가 표시됩니다. 이 출처에 포함되면 클릭률이 기존 검색 결과보다 훨씬 높습니다. 포함되지 않으면 AI 답변 아래로 밀려서 거의 보이지 않습니다.

GEO는 이 출처 3~5개 안에 들어가는 것을 목표로 합니다.

## GEO에서 AI가 참조하는 3가지 시그널

### 1. 권위 시그널

AI는 "이 사이트가 이 주제에 대해 전문성이 있는가"를 판단합니다. 판단 기준:

- 구조화 데이터에 회사 정보, 서비스 유형, 대표자 정보가 명시되어 있는가
- 해당 주제에 대한 콘텐츠가 사이트에 충분히 있는가
- 다른 사이트에서 이 사이트를 인용하고 있는가

구조화 데이터가 없으면 AI는 사이트의 전문성을 파악하지 못합니다. 코드로 명시적으로 알려줘야 합니다.

### 2. 답변 적합성

AI는 사용자의 질문에 직접 답변하는 콘텐츠를 선호합니다.

- 질문-답변 형태의 콘텐츠가 있는가
- 답변이 명확하고 구체적인가 (숫자, 사례 포함)
- 답변이 코드상에서 보이는 상태인가 (숨겨져 있으면 읽지 못함)

사이트에 잘 구성된 질문-답변 섹션이 있으면, AI가 답변을 생성할 때 직접 인용합니다.

### 3. 기술 신뢰도

AI는 기술적으로 잘 만들어진 사이트를 더 신뢰합니다.

- 페이지 속도가 빠른가
- 모바일에서 잘 작동하는가
- 보안 인증서(HTTPS)가 적용되어 있는가
- llms.txt 파일이 있는가

느리고, 깨지고, 보안이 안 된 사이트는 AI가 참조 대상에서 제외합니다.

## GEO를 적용하는 방법

기존 SEO와 AEO 위에 GEO를 추가로 적용합니다.

**이미 해야 하는 것 (SEO + AEO):**
- 구조화 데이터 설계
- llms.txt 파일 생성
- 시맨틱 코드 구조
- 속도 최적화

**GEO를 위해 추가로 해야 하는 것:**
- 업종별 핵심 질문 10개에 대한 명확한 답변 콘텐츠
- 사이트 내 콘텐츠 간 내부 링크 강화 (주제별 연결)
- 대표자/저자 정보를 구조화 데이터에 명시
- 블로그 글에 구체적 숫자와 사례 포함

아닙니다. GEO는 SEO와 AEO 위에 쌓이는 세 번째 층입니다. SEO가 기반이고, AEO가 확장이고, GEO가 최신 층입니다. 셋 다 해야 합니다.

**빌더(아임웹, 카페24)로 만든 사이트에서 GEO를 적용할 수 있나요?**
구조적으로 어렵습니다. GEO의 핵심인 구조화 데이터 정밀 설계, llms.txt 생성, 시맨틱 코드 구조는 [코드 기반 사이트](/services/website)에서만 가능합니다.

## 블로그를 쓰는 도구와 보여주는 사이트는 분리할 수 있다

전통적인 사이트는 글을 쓰는 관리자 페이지와 글을 보여주는 사이트가 한 시스템 안에 있습니다. 아임웹·카페24·워드프레스 모두 같은 구조입니다. 글을 쓸 때마다 관리자 페이지에 로그인해 자체 에디터를 써야 합니다.

Headless CMS는 이 두 영역을 분리합니다. 글은 노션·Sanity·Strapi 같은 별도 도구로 쓰고, 사이트는 그 데이터를 API로 불러와 자동으로 보여줍니다. 결과적으로 운영자는 평소 쓰던 도구로 글만 쓰면 됩니다.

---

## Headless CMS의 3가지 작동 방식

**1. 노션(Notion) 백엔드:** 노션 페이지에 글을 작성. 노션 API를 통해 사이트가 데이터를 가져와 자동으로 HTML로 변환. 노션의 데이터베이스 기능으로 글 목록·태그·발행일을 관리.

**2. Sanity Studio:** 글 작성에 특화된 오픈소스 CMS. 이미지 자동 최적화, 다국어 번역, 버전 관리, 권한 관리가 표준 탑재. 중대형 사이트에 적합.

**3. Strapi·Contentful:** API 우선 설계의 헤드리스 CMS. 사이트 규모가 크고 다수의 콘텐츠 타입(블로그+제품+이벤트)을 관리할 때 적합.

치로웹디자인은 사이트 규모와 운영 인원에 따라 위 셋 중 적합한 도구를 골라 적용합니다.

## 빌더 에디터의 한계

아임웹·카페24의 자체 에디터는 다음 문제를 안고 있습니다.

- **인라인 스타일 누적:** 외부에서 텍스트를 복사·붙여넣으면 `<span style="font-family: ...">` 같은 잡 코드가 쌓여 SEO를 망침
- **마크다운 미지원:** 글쓰기 속도가 떨어지고, 다른 문서 시스템과 호환되지 않음
- **버전 관리 부재:** 글을 잘못 수정했을 때 이전 버전으로 복원하기 어려움
- **이미지 최적화 부재:** 업로드한 원본이 그대로 노출되어 페이지 무거워짐
- **백업·이관 어려움:** 글을 한꺼번에 내려받거나 다른 시스템으로 옮기는 작업이 매우 까다로움

매번 같은 작업을 반복하다 보면 1년에 수십 시간이 사라집니다.

## Headless CMS의 운영 흐름

치로웹디자인 블로그를 예로 들면 다음 흐름으로 운영됩니다.

1. 노션 데이터베이스에 글을 작성. 제목·본문·태그·발행일 입력
2. "상태"를 "발행 예정"으로 변경
3. GitHub Actions가 1시간마다 노션 API를 호출해 새 글을 감지
4. 새 글을 MDX 파일로 변환해 저장소에 자동 커밋
5. Vercel이 자동 배포. 5분 안에 사이트에 노출

운영자는 1단계와 2단계만 합니다. 나머지는 자동입니다.

## "글쓰기에만 집중할 수 있는 환경"이 콘텐츠 양을 결정한다

운영자가 매번 잡 작업에 시간을 쓰면 콘텐츠 발행 빈도가 떨어집니다. 노션에서 글만 쓰면 자동으로 게시되는 환경에서는 발행 속도가 평균 2~3배 빨라집니다. 콘텐츠 양은 검색 트래픽과 직접 비례하므로 매출에 누적됩니다.

치로웹디자인은 모든 사이트에 Headless CMS 연동을 기본 옵션으로 제공합니다. 풀스택 기반이기 때문에 가능한 구조입니다. [치로의 홈페이지 제작 서비스](/services/website)에서 적용 사례를 확인할 수 있습니다. 빌더에서 풀스택으로 전환하는 절차는 [빌더에서 커스텀 전환 가이드](/blog/builder-to-custom-switch-guide)를 참고하세요.

## 자주 묻는 질문

**노션을 블로그 백엔드로 쓰면 보안 문제가 없나요?**
노션은 외부에 공개되는 권한과 API로만 읽히는 권한이 분리됩니다. 사이트는 API 권한으로만 접근하므로 노션 페이지 자체는 외부에 노출되지 않습니다.

**기존 빌더 블로그 글을 노션·Sanity로 옮길 수 있나요?**
가능합니다. 자동 스크래퍼로 기존 글을 한꺼번에 가져와 새 시스템에 등록합니다. 글 100~500개 기준 1~3일 작업입니다.

**Headless CMS를 쓰면 운영 비용이 더 드나요?**
노션은 기존에 이미 쓰고 있다면 추가 비용이 없습니다. Sanity는 월 0~100달러, Strapi는 자체 호스팅 시 무료입니다. 빌더 구독료(월 2~10만원)와 비교하면 비슷하거나 더 저렴한 구간이 많습니다.

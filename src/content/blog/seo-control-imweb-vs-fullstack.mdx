## 빌더가 제공하는 SEO 항목은 단 4가지

아임웹·카페24가 SEO 메뉴에서 사용자에게 허용하는 입력 항목은 페이지 타이틀, 메타 디스크립션, OG 이미지, 키워드 메타태그 4가지가 전부입니다. 이외의 모든 SEO 신호는 플랫폼이 자동으로 생성하거나 아예 생성하지 않습니다. 사용자는 손댈 권한이 없습니다.

문제는 2026년 기준 구글이 검색 순위 결정에 활용하는 SEO 신호가 200개 이상이라는 점입니다. 4개를 입력했다고 검색에 잡히는 시대가 아닙니다.

---

## 풀스택 홈페이지가 통제하는 SEO 신호

Next.js·Remix 같은 풀스택 프레임워크는 서버 사이드 렌더링(SSR)과 메타데이터 API를 통해 페이지마다 다음을 직접 설계합니다.

- **메타 태그 전체:** title, description, OG, Twitter Card, robots, canonical, hreflang
- **JSON-LD 구조화 데이터:** Article, FAQPage, HowTo, BreadcrumbList, Organization, Service, Product, Review 등 페이지 성격에 맞춰 다중 적용
- **시맨틱 URL:** `/blog/aeo-vs-seo-difference` 형태로 키워드를 URL에 포함
- **동적 sitemap.xml:** 새 글이 발행되면 자동으로 사이트맵에 추가
- **AI 크롤러 전용 robots 설정:** GPTBot, ClaudeBot, PerplexityBot 개별 허용
- **llms.txt 루트 파일 배치:** AI 검색 인용을 위한 신호

## 비교표

| SEO 항목 | 아임웹·카페24 | 풀스택(Next.js) |
|---------|------------|----------------|
| 메타 태그 | 4종 입력 | 무제한 |
| JSON-LD | 일부 자동 | 8종 이상 직접 설계 |
| URL 구조 | 플랫폼 고정 | 100% 자유 |
| 동적 sitemap | 자동(편집 불가) | 코드로 제어 |
| AI 크롤러 허용 | 불가 | 개별 설정 |
| llms.txt | 생성 불가 | 기본 포함 |

## "제어권"이 검색 순위를 결정한다

같은 글을 아임웹과 풀스택에 동시 발행해도 검색 결과가 다릅니다. 풀스택 쪽은 리치 스니펫이 붙고, FAQ가 펼쳐지고, 별점·이미지가 함께 노출됩니다. 아임웹 쪽은 파란색 제목 한 줄로만 표시됩니다. 클릭률은 평균 2~3배 차이가 납니다.

이 차이는 콘텐츠 품질이 아니라 **HTML 구조와 메타 신호의 차이**에서 옵니다. 빌더는 이 영역을 사용자에게 열어주지 않습니다. 풀스택은 페이지 단위로 100% 통제합니다.

치로웹디자인은 모든 페이지의 SEO 신호를 디렉터가 직접 설계합니다. 글의 성격에 따라 JSON-LD를 다르게 심고, 메타 태그를 페이지별로 분리합니다. [치로의 홈페이지 제작 서비스](/services/website)에서 작업 기준을 확인할 수 있습니다. 빌더 사이트의 SEO 한계는 [빌더 vs 커스텀 코딩 비교](/blog/builder-vs-custom-coding)에서 더 다룹니다.

## 자주 묻는 질문

**아임웹 유료 플랜을 쓰면 SEO 항목이 더 풀리나요?**
타이틀·디스크립션 외에 추가로 열리는 항목은 거의 없습니다. 구조화 데이터, robots.txt 세부 설정, llms.txt 같은 항목은 플랜과 무관하게 차단됩니다. 플랜 가격은 기능 제한과 직접 관계가 없습니다.

**카페24에서 SEO를 잘하면 풀스택과 비슷한 결과가 나오나요?**
기본 노출 수준까지는 가능합니다. 다만 리치 스니펫, AI 검색 인용, Core Web Vitals 상위 점수는 구조적으로 어렵습니다. 경쟁 키워드 상위 노출이 목표라면 풀스택이 필수입니다.

**SEO 항목을 100가지나 통제할 필요가 정말 있나요?**
모든 페이지에 200가지를 다 적용할 필요는 없습니다. 다만 페이지 성격에 따라 어떤 신호를 얼마나 줄지 선택할 수 있느냐가 중요합니다. 빌더는 이 선택권 자체를 차단합니다.

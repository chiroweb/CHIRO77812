## 상품 페이지 1만 개짜리 쇼핑몰이 구글에서 안 보이는 이유

상품이 많을수록 SEO에 유리해야 한다는 생각은 틀렸습니다. 구조화되지 않은 상품 페이지 1만 개는 구글이 읽기 어렵고, 중복 콘텐츠로 처리되어 오히려 순위를 낮춥니다.

2026년 구글 쇼핑 데이터에서 상위 10위 안에 드는 상품 페이지의 97%는 Product 스키마를 포함합니다. 리뷰 스키마까지 적용한 경우 클릭률은 미적용 대비 평균 35% 높습니다.

쇼핑몰 SEO는 블로그 SEO와 다릅니다. 콘텐츠의 양보다 상품 페이지 하나하나의 구조적 완성도가 핵심입니다.

## Product 스키마가 만드는 리치 결과

구조화 데이터는 검색엔진에게 페이지 내용을 코드로 명시하는 방법입니다. Product 스키마를 적용하면 구글 검색 결과에 다음 정보가 추가됩니다.

- 가격과 재고 상태 (In Stock / Out of Stock)
- 별점과 리뷰 수
- 배송 정보
- 특가/할인 표시

이 정보가 검색 결과에 보이면 방문자는 클릭 전에 이미 구매 결정의 80%를 끝냅니다. 경쟁 상품과 같은 검색 결과 페이지에서 별점 4.8(리뷰 234개)이 표시된 상품이 텍스트만 있는 상품보다 유리한 것은 명확합니다.

Product 스키마에 반드시 포함해야 하는 항목은 다음과 같습니다.

| 항목 | 설명 |
|------|------|
| name | 상품명 |
| description | 상품 설명 (150자 이상 권장) |
| image | 고화질 상품 이미지 URL |
| offers | 가격, 통화, 재고 상태 |
| aggregateRating | 별점 평균, 리뷰 수 |
| brand | 브랜드명 |

## 리뷰 스키마와 UGC의 SEO 가치

리뷰는 쇼핑몰 SEO에서 두 가지 역할을 합니다.

첫째, 구조화된 리뷰 데이터(Review 스키마)는 검색 결과에 별점을 표시합니다. 리뷰 스키마가 없으면 실제 리뷰가 수백 개여도 구글 검색 결과에는 아무것도 보이지 않습니다.

둘째, 리뷰 텍스트는 상품 페이지의 키워드 다양성을 높입니다. 판매자가 작성한 상품 설명에 없는 표현을 구매자 리뷰가 채웁니다. "실제 착용감", "세탁 후 변화", "키 172cm 기준 사이즈" 같은 구체적인 표현이 롱테일 검색에 기여합니다.

리뷰를 SEO 자산으로 만들려면 구조화 데이터가 필수입니다. 리뷰를 이미지로만 표시하거나 자바스크립트로만 렌더링하면 구글이 읽지 못합니다.

## 구조화된 URL과 카테고리 설계

쇼핑몰 URL은 상품과 카테고리 구조를 그대로 반영해야 합니다.

**잘못된 구조**: `/product?id=38291&cat=5`
**올바른 구조**: `/category/남성의류/반팔티셔츠/product-name`

올바른 URL 구조는 두 가지 효과를 만듭니다. 검색엔진이 카테고리와 상품의 관계를 파악합니다. 카테고리 페이지 자체가 "남성 반팔티셔츠"라는 키워드로 검색 노출 기회를 갖습니다.

카테고리 페이지 SEO는 상품 페이지 SEO만큼 중요합니다. 카테고리 페이지에도 고유한 설명 텍스트, 메타 태그, 내부 링크 구조가 필요합니다. 카테고리 페이지를 단순히 상품 목록으로만 쓰면 검색 노출 기회를 절반 잃는 것과 같습니다.

## 네이버 쇼핑 vs 구글 쇼핑의 차이

국내 쇼핑몰 운영자에게 두 채널은 전략이 다릅니다.

**네이버 쇼핑**은 가격, 리뷰 수, 판매량이 노출 순위에 직접 영향을 줍니다. 네이버 쇼핑은 네이버 스마트스토어 또는 쇼핑 파트너 연동이 전제입니다. 자체 쇼핑몰만으로는 네이버 쇼핑에 노출되지 않습니다.

**구글 쇼핑**은 구글 머천트 센터에 상품 피드를 등록하면 Google Shopping 탭과 일반 검색 결과에 노출됩니다. Product 스키마 + 구글 머천트 센터 피드의 조합이 최적입니다. 구글 쇼핑은 국내 시장보다 해외 고객 유입에 특히 효과적입니다.

자체 쇼핑몰의 SEO 이점은 두 플랫폼 의존도 없이 구글 자연 검색 트래픽을 직접 받을 수 있다는 점입니다. 네이버 스마트스토어나 쿠팡에서는 플랫폼 로직 변경 시 노출이 갑자기 감소할 수 있습니다. 자체 도메인 기반 쇼핑몰은 SEO 자산이 축적될수록 안정성이 높아집니다.

## 상품 설명 최적화의 실제 기준

상품 설명은 SEO와 구매 전환을 동시에 충족해야 합니다. 두 가지 목표가 상충하지 않습니다.

SEO 관점에서 상품 설명에 필요한 것: 메인 키워드 1~2회 자연스럽게 포함, 연관 키워드(색상, 소재, 용도), 300자 이상의 고유한 텍스트(제조사 설명 그대로 복사 금지).

전환 관점에서 상품 설명에 필요한 것: 구매자의 질문에 대한 답(사이즈 기준, 소재 설명, 관리 방법), 사용 상황 설명(어떤 상황에서 사용하는지), 구체적인 수치(무게, 크기, 용량).

같은 상품을 제조사 설명 그대로 올린 페이지는 "중복 콘텐츠"로 처리되어 검색 순위가 낮아집니다. 고유한 설명을 작성하는 것이 SEO와 전환 모두에서 유리합니다.

## 자주 묻는 질문

**카페24, 메이크샵으로 만든 쇼핑몰에도 Product 스키마를 적용할 수 있나요?**
일부 템플릿에서 제한적으로 가능하지만 정밀한 커스터마이징은 어렵습니다. 상품 수가 1,000개 이상이면 자동화된 스키마 생성 구조가 필요한데, 이는 커스텀 개발 환경에서만 효율적으로 구현됩니다.

**상품 페이지 수가 많을수록 SEO에 불리한가요?**
구조가 나쁘면 불리합니다. URL 체계, 메타 태그 자동 생성, 중복 콘텐츠 관리가 설계되어 있으면 상품 수가 많을수록 롱테일 검색 노출 기회가 늘어납니다.

**재고 없는 상품 페이지를 삭제해야 하나요?**
삭제보다 재고 상태를 "Out of Stock"으로 표시하고 페이지를 유지하는 것이 SEO에 유리합니다. 페이지를 삭제하면 해당 URL로 쌓인 SEO 자산이 사라집니다. Product 스키마에서 재고 상태를 명확히 표시하면 됩니다.

**구글 서치 콘솔에서 "상품 구조화 데이터 오류"가 보이면 어떻게 해야 하나요?**
필수 항목 누락이 가장 흔한 원인입니다. name, offers(price + availability + priceCurrency)가 없으면 오류가 발생합니다. 구글의 리치 결과 테스트 도구(search.google.com/test/rich-results)에서 개별 URL을 진단하면 구체적인 오류 항목을 확인할 수 있습니다.

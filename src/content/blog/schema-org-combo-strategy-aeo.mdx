## FAQPage 스키마 하나만 적용한 사이트는 AEO에서 불리하다

대부분의 AEO 가이드가 FAQPage 스키마 적용으로 끝납니다. 부족합니다. AI 답변 분석에서 인용되는 사이트들은 평균 4.2종의 Schema.org 스키마를 동시에 사용합니다. 단일 스키마만 적용한 사이트는 인용률이 평균의 1/3 수준이었습니다.

치로웹디자인이 AEO 표준 패키지에 포함하는 Schema.org 8종과 페이지 유형별 조합 전략을 정리합니다.

---

## 핵심 8종 스키마

| 스키마 | 용도 | 적용 페이지 |
|------|------|---------|
| Organization | 회사 정보 | 전체 페이지 |
| WebSite | 사이트 메타 | 전체 페이지 |
| BreadcrumbList | 페이지 경로 | 전체 페이지 |
| Service | 서비스 설명 | 서비스 페이지 |
| FAQPage | 자주 묻는 질문 | 모든 핵심 페이지 |
| Article | 블로그 글 | 블로그 글 |
| HowTo | 단계별 가이드 | 가이드 글 |
| LocalBusiness | 지역 비즈니스 | 회사 소개·연락처 |

이 8종을 각 페이지 유형에 맞게 조합해 삽입합니다.

---

## 페이지 유형별 조합 전략

홈페이지: Organization + WebSite + BreadcrumbList. 사이트 전체 메타 정보를 담습니다. 검색엔진과 AI에 "이 사이트가 무엇이고 누가 운영하는가"를 1차로 알립니다.

서비스 페이지: Service + FAQPage + BreadcrumbList. Service 스키마는 가격, 제공 기간, 서비스 영역을 구조화합니다. FAQPage는 해당 서비스에 대한 자주 묻는 질문을 마크업합니다.

블로그 글: Article + FAQPage + BreadcrumbList. Article 스키마는 저자, 발행일, 수정일, 헤드라인을 명시합니다. AI Overview 인용 조건에서 [발행일과 수정일 메타데이터](/blog/google-ai-overview-citation-factors)의 가산점을 가져옵니다.

가이드 글: HowTo + FAQPage + BreadcrumbList. HowTo 스키마는 단계, 도구, 예상 소요 시간을 구조화합니다. AI는 HowTo 스키마를 단계별 답변에 그대로 변환합니다.

회사 소개·연락처: LocalBusiness + Organization + BreadcrumbList. LocalBusiness 스키마는 지역 검색과 지도 검색에 필수입니다. 네이버 플레이스, 구글 비즈니스 프로필 데이터와 일관되게 작성해야 합니다.

---

## Schema 중복 사용의 함정

같은 페이지에 같은 스키마를 두 번 삽입하면 구글이 무효 처리합니다. Article 스키마는 페이지당 1개만 허용됩니다. FAQPage 스키마도 페이지당 1개입니다. 다만 다른 스키마는 한 페이지에 여러 개 공존 가능합니다.

JSON-LD 블록을 분리하는 것이 안전합니다. 한 `<script type="application/ld+json">` 안에 모든 스키마를 묶지 말고, 스키마 종류마다 별도 블록으로 작성합니다. 일부 스키마에 오류가 있어도 다른 스키마는 정상 작동합니다.

---

## 구글 리치 결과 vs AEO에서의 가치 차이

스키마별로 구글 리치 결과 노출 효과와 AEO 인용 효과가 다릅니다.

| 스키마 | 구글 리치 결과 | AEO 인용 |
|------|----------|---------|
| FAQPage | 노출 줄어듦 (2024 정책 변경) | 매우 중요 |
| HowTo | 노출 없음 (2023 폐지) | 중요 |
| Article | 일부 노출 | 중요 |
| LocalBusiness | 노출 | 중요 |
| Service | 노출 적음 | 중요 |

FAQPage와 HowTo는 구글 검색 결과에서 표시가 줄었지만 AEO에서는 여전히 핵심 신호입니다. SEO 관점에서 가치가 떨어졌다고 빼면 안 됩니다.

## Schema 검증과 모니터링

스키마 삽입 후 검증은 필수입니다. 오류가 있는 스키마는 효과가 없을 뿐 아니라 사이트 신호를 약하게 만듭니다.

검증 도구는 두 가지를 함께 사용합니다. 구글 리치 결과 테스트(search.google.com/test/rich-results)와 Schema Markup Validator(validator.schema.org). 두 도구에서 모두 통과해야 합니다.

발행 후 구글 서치 콘솔의 "강화" 탭에서 스키마 인식 현황을 매주 확인합니다. 오류가 누적되면 도메인 신뢰도가 떨어집니다. 치로웹디자인의 [AEO 자동화 서비스](/services/seo-aeo)는 스키마 자동 검증과 월간 모니터링을 포함합니다.

## 자주 묻는 질문

**스키마를 너무 많이 넣으면 역효과가 있나요?**
관련 없는 스키마를 넣으면 구글이 스팸으로 판단할 수 있습니다. 페이지 콘텐츠와 일치하는 스키마만 삽입해야 합니다. 8종 모두 적용 가능한 페이지는 거의 없습니다. 페이지 유형별로 3~5종이 적정 범위입니다.

**스키마는 직접 작성하나요, 플러그인을 쓰나요?**
워드프레스는 Rank Math, Yoast SEO Premium이 일부 스키마를 자동 생성합니다. Next.js·React 기반 커스텀 사이트는 JSON-LD를 페이지별로 직접 삽입하는 것이 정확합니다. 직접 삽입이 가장 통제력이 높습니다.

**스키마 적용 후 효과는 언제 보이나요?**
구글 리치 결과 반영은 2~6주, AI 검색 인용 증가는 3~8주가 일반적입니다. 도메인 권위가 강한 사이트일수록 반영이 빠릅니다. 신규 도메인은 2~3개월 누적이 필요합니다.

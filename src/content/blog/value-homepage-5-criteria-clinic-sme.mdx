## 가성비 홈페이지의 진정한 의미

200만~300만원대 홈페이지의 가성비를 판단하는 기준은 초기 제작비가 아닙니다. 3년 뒤 누가 더 많은 고객을 데려오고, 운영 비용이 얼마나 적은지로 결정됩니다. 같은 가격대에서도 선택에 따라 월간 수익이 50만원 차이 날 수 있습니다.

홈페이지는 한 번 선택하면 최소 3년 이상 사용하는 자산입니다. 초기 구매 가격보다 중요한 것은, 그 홈페이지가 365일 일하는 세일즈맨이 되는가 하는 점입니다.

## 1. SEO 권한 — 누가 검색 결과를 통제하는가

**당신의 사이트가 구글에서 노출되려면, 당신이 SEO를 통제해야 합니다.**

아임웹이나 카페24 같은 빌더는 URL 구조를 통제합니다. `/page?id=123` 같은 형태입니다. 구글은 이런 구조의 사이트를 홈페이지만큼 중요하게 취급하지 않습니다. 반면 치로는 `/clinic-information` 같은 명확한 URL을 당신이 원하는 대로 설정합니다.

더 중요한 것은 meta 태그입니다. 각 페이지의 제목, 설명, 구조화 데이터를 당신의 키워드에 맞춰 최적화할 수 있어야 합니다. 빌더는 일부 통제만 허용하고, 코드 기반 사이트는 완전한 통제가 가능합니다.

| 항목 | 빌더 (아임웹) | 코드 기반 (치로) |
|------|----------|----------|
| URL 구조 통제 | 제한됨 | 완전 통제 |
| meta 태그 커스텀 | 부분만 가능 | 모든 페이지 개별 설정 |
| Schema 마크업 | 기본만 제공 | 100% 커스텀 |
| 검색 결과 노출 속도 | 4~12주 | 2~6주 |

당신의 사이트가 구글에서 "의원명 + 진료과목" 검색에 1페이지에 뜨려면, 이 기준들이 필수입니다.

## 2. 코드 소유권 — 평생 종속인가, 독립인가

**가장 위험한 계약은 "호스팅은 우리가 관리해드립니다"라는 약속입니다.**

많은 에이전시가 자신들의 서버에 당신의 사이트를 올려놓고, 매달 호스팅비를 청구합니다. 이 경우 다음과 같은 문제가 발생합니다.

1. 계약이 끝나면 사이트를 가져갈 수 없습니다
2. 호스팅비는 계속 청구되고 (보통 월 10~30만원), 올려주지 않으면 사이트가 내려갑니다
3. 다른 곳으로 이전하려 해도 소스 코드를 주지 않습니다

치로는 Vercel에 배포하고, 깃허브에 코드를 저장해줍니다. 당신의 계정에 모든 권한이 있고, 언제든 다른 곳으로 이전할 수 있습니다. 이것이 진정한 자산입니다.

## 3. 성능 — 로딩 속도가 매출을 결정한다

**로딩에 3초 이상 걸리면 방문자의 53%가 떠납니다.**

구글도 이를 알고 있어서, Core Web Vitals 점수가 낮은 사이트의 검색 순위를 직접 내립니다. 같은 콘텐츠인데 하나는 Lighthouse 점수 90점, 다른 하나는 50점이면, 결국 느린 사이트가 뒤로 밀립니다.

아임웹으로 만든 사이트의 평균 Lighthouse 점수는 50~70점대입니다. 추적 코드, 광고 스크립트가 많아질수록 더 떨어집니다. 치로 사이트의 평균 점수는 85~95점입니다.

성능이 좋으면 다음을 얻습니다.
- 검색 순위 상승 (2~3등급 가능)
- 방문자 이탈률 감소
- 모바일 전환율 증가
- 서버 비용 절감

## 4. AEO — ChatGPT도 당신의 사이트를 인용하는가

**2026년, 검색 트래픽의 25%가 이미 AI 검색 (ChatGPT, Perplexity)으로 이동했습니다.**

구글 검색만으로는 부족합니다. ChatGPT가 "의원 추천" 같은 질문에 당신의 사이트를 인용하지 않으면, 그 고객을 놓칩니다.

AI가 당신의 사이트를 인용하려면, 세 가지가 필요합니다.

1. **robots.txt에 AI 크롤러 허용** — llms.txt 파일 포함
2. **구조화 데이터** — FAQPage, LocalBusiness, BreadcrumbList 스키마
3. **명확한 답변 구조** — "Q&A 섹션"으로 명확히 구분

빌더는 이 세 가지를 모두 구현하기 어렵습니다. 코드 기반이라면 쉽습니다.

## 5. 운영비 — 유지비로 얼마나 더 낼 것인가

**초기 비용 외에 계속 내야 하는 비용을 확인하세요.**

| 항목 | 빌더 | 코드 기반 |
|------|------|---------|
| 초기 제작비 | 180~250만원 | 200~300만원 |
| 월간 호스팅비 | 10~20만원 | 0~2만원 (Vercel 무료~저가) |
| 연간 유지보수 | 무료 | 무료 또는 시간비 |
| 3년 총 비용 | 396~540만원 | 236~372만원 |

같은 초기 비용이어도, 3년 뒤 총 비용은 100만원 이상 차이 날 수 있습니다.

## 자주 묻는 질문

### Q. 200만~300만원대에서 정말 위의 5가지를 모두 확인할 수 있나요.

네. 이 기준들은 가격과 무관하게 필수입니다. 다만 기능이나 디자인의 복잡도에 따라 비용이 결정됩니다. 기본적인 병원·의원 홈페이지라면 200만~250만원대면 충분합니다. 전문직(변호사, 세무사) 포트폴리오나 제조업 카탈로그라면 250~300만원대입니다.

### Q. 빌더로 만든 사이트도 SEO를 할 수 있지 않나요.

부분적으로 가능합니다. 하지만 한계가 있습니다. URL 구조 변경은 불가능하고, meta 태그는 제한된 범위만 수정 가능합니다. 코드 기반이면 이런 제약이 없습니다.

### Q. 호스팅비 없다는 게 정말 가능한가요.

가능합니다. Vercel은 일반 트래픽 수준의 사이트는 무료로 제공합니다. 월 100만 방문 이상의 대규모 사이트도 보통 월 2~5만원대입니다. 이는 기존 호스팅비의 1/5 수준입니다.

### Q. 코드 소유는 실제로 언제 필요한가요.

3년~5년 뒤입니다. 에이전시와 계약이 끝나면, 새로운 디자이너나 개발자에게 사이트를 맡길 때 당신이 코드를 제공할 수 있어야 합니다. 빌더 기반이면 이것이 불가능합니다.

## 두 방식의 차이는 "얼마나 자유로운가"에서 결정된다

아임웹, 카페24, Wix로 만든 사이트와 Next.js, React로 직접 코딩한 사이트는 겉보기에 비슷해 보입니다. 하지만 검색엔진이 보는 방식, AI 검색 대응 능력, 속도, 확장성에서 구조적 차이가 있습니다.

2026년 현재 구글과 ChatGPT, Perplexity 같은 AI 검색의 트래픽 비중이 빠르게 늘고 있습니다. 검색 트래픽의 25% 이상이 AI 기반 검색에서 발생하는 시점에서, 어떤 방식으로 만드느냐는 비즈니스 성과에 직접 영향을 줍니다.

---

## 빌더란 무엇인가

빌더는 코딩 없이 홈페이지를 만들 수 있는 플랫폼입니다. 아임웹, 카페24, Wix, Squarespace가 대표적입니다.

드래그 앤 드롭으로 레이아웃을 구성하고, 제공되는 템플릿을 선택한 뒤 텍스트와 이미지를 교체하면 홈페이지가 만들어집니다. 서버 설정, 코드 배포, 호스팅 관리를 신경 쓰지 않아도 됩니다.

장점은 명확합니다. 빠르고, 초기 비용이 낮고, 기술 지식이 없어도 됩니다. 단점도 명확합니다. 플랫폼이 허용하는 범위 안에서만 모든 것을 결정해야 합니다.

---

## 커스텀 코딩이란 무엇인가

커스텀 코딩은 처음부터 코드를 작성해서 사이트를 만드는 방식입니다. Next.js, React, Vue.js 같은 기술 스택을 사용하고, Vercel, AWS, Netlify 같은 플랫폼에 배포합니다.

모든 코드를 직접 작성하므로 구조적 자유도가 높습니다. 어떤 태그를 쓸지, 데이터를 어떻게 구조화할지, 서버를 어떻게 구성할지 모두 결정할 수 있습니다.

초기 제작 시간과 비용이 더 필요하지만, 이후 추가 개발, 검색 최적화, 기능 확장이 훨씬 수월합니다.

---

## 5가지 기준으로 비교

### 1. 페이지 속도

구글 Lighthouse 기준으로 빌더 사이트의 평균 점수는 50~75점입니다. 커스텀 코딩 사이트는 90~98점을 기록합니다.

빌더가 느린 이유는 "모든 사람을 위한 코드"가 포함되어 있기 때문입니다. 쇼핑몰 기능이 필요 없어도 쇼핑몰 관련 코드가 함께 로딩됩니다. 커스텀 코딩에서는 필요한 코드만 포함할 수 있습니다.

구글은 Core Web Vitals(핵심 웹 지표)를 검색 순위 신호로 사용합니다. 속도 차이는 검색 순위 차이로 직결됩니다.

### 2. SEO 구조

커스텀 코딩에서는 URL 구조를 원하는 대로 설계할 수 있습니다. `/services/seo-aeo` 같은 의미 있는 주소가 가능합니다.

빌더에서는 플랫폼이 정한 URL 구조를 따라야 합니다. 아임웹은 `/page/23`, 카페24는 `/board/view.html?no=123` 같은 형태가 기본입니다.

구조화 데이터(JSON-LD)도 마찬가지입니다. 서비스 페이지에는 Service 스키마, FAQ 페이지에는 FAQPage 스키마를 넣어야 구글 검색 결과에 리치 스니펫이 표시됩니다. 빌더에서는 이 작업이 제한적이거나 불가능합니다.

### 3. AEO(AI 검색 최적화) 대응

AEO는 ChatGPT, Perplexity, Google AI Overviews 같은 AI 검색에서 인용되기 위한 최적화입니다.

AI 검색 대응에 필요한 핵심 파일 중 하나가 llms.txt입니다. 이 파일은 사이트 루트 경로에 있어야 하는데, 빌더에서는 루트에 임의 파일을 생성할 수 없습니다.

또한 AI 로봇(GPTBot, ClaudeBot, PerplexityBot)을 명시적으로 허용하는 robots.txt 설정도 필요합니다. 빌더에서는 이 설정을 세밀하게 제어할 수 없습니다.

커스텀 코딩에서는 llms.txt, robots.txt, 구조화 데이터, 시맨틱 HTML 모두 처음부터 AEO 기준에 맞게 설계합니다.

### 4. 확장성

비즈니스가 성장하면 홈페이지에 새 기능이 필요해집니다. 예약 시스템, 회원 전용 콘텐츠, API 연동, 자동화된 콘텐츠 관리 등이 그 예입니다.

빌더에서 이런 기능을 추가하려면 플랫폼이 제공하는 앱이나 플러그인을 써야 합니다. 원하는 기능이 없으면 구현 자체가 불가능합니다. 여러 플러그인을 사용하면 속도가 더 느려지는 문제도 생깁니다.

커스텀 코딩에서는 모든 기능을 직접 개발하거나 외부 API와 연동할 수 있습니다. 확장 범위에 제한이 없습니다.

### 5. 코드 소유권

빌더 사이트는 플랫폼을 떠나면 사라집니다. 아임웹을 해지하면 사이트가 없어집니다. 플랫폼이 서비스를 종료하거나 가격 정책을 바꾸면 선택권이 없습니다.

커스텀 코딩에서는 코드 전체가 고객 소유입니다. 납품 이후 어떤 호스팅에 올릴지, 어떤 에이전시와 일할지 자유롭게 선택할 수 있습니다.

## 어떤 상황에서 빌더를 써도 되는가

빌더가 적합한 경우가 있습니다. 온라인 존재감만 필요하고 검색 유입이 목표가 아닐 때, 단기 프로젝트나 이벤트 페이지, 아직 비즈니스 방향이 확정되지 않은 초기 단계입니다.

검색에서 발견되어야 하고, AI 검색에서 인용되어야 하고, 장기적으로 유지하고 성장시킬 사이트라면 커스텀 코딩이 맞는 선택입니다.

치로는 코드 기반으로만 사이트를 만듭니다. 빌더 기반 제작은 하지 않습니다. 처음부터 검색 구조를 고려해서 설계하기 때문입니다. [치로의 홈페이지 제작 서비스](/services/website)에서 작업 방식을 확인할 수 있습니다.

빌더 기반 사이트에서 코드 기반으로 전환하는 것은 [아임웹·카페24 SEO 한계](/blog/imweb-cafe24-seo-limits) 글에서 더 자세히 다루고 있습니다.

## 자주 묻는 질문

**아임웹으로 만들어도 구글에 나올 수 있나요?**
노출 자체는 됩니다. 하지만 구조화 데이터, llms.txt, 속도 최적화가 불가능하기 때문에 경쟁 키워드에서 상위 노출은 어렵습니다. AI 검색(ChatGPT, Perplexity)에서 인용될 가능성도 거의 없습니다.

**커스텀 코딩은 얼마나 비싼가요?**
규모와 기능에 따라 다릅니다. 치로 기준으로 기본 기업 홈페이지는 99만원부터 시작합니다. 빌더보다 초기 비용이 높지만, 구독 비용이 없고 장기적으로 검색 성과가 누적된다는 점에서 비용 대비 효과가 더 높습니다.

**직접 내용을 수정하려면 코딩을 알아야 하나요?**
아닙니다. 커스텀 코딩 사이트에도 CMS(콘텐츠 관리 시스템)를 연결하면 텍스트, 이미지, 블로그 글 수준의 수정은 관리자 페이지에서 직접 할 수 있습니다.

**빌더 사이트를 커스텀 코딩으로 전환하면 기존 SEO 자산을 잃나요?**
리디렉션 설정을 올바르게 하면 기존 검색 자산을 보존할 수 있습니다. 오히려 구조 개선으로 인해 이전보다 순위가 오르는 경우가 더 많습니다.

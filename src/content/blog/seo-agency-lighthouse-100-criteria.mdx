## 100점은 만드는 것보다 유지가 어렵다

2026년 현재, 한국에서 "SEO 에이전시 추천" 키워드로 검색하면 100곳 이상의 업체가 나옵니다. 이 중 자사 클라이언트 사이트가 운영 중에도 라이트하우스 100점을 유지한다는 것을 라이브 점수로 증명하는 곳은 5% 미만입니다.

100점은 '한 번' 만드는 것과 '운영 중 유지'가 완전히 다른 작업입니다.

## 1회성 100점이 무너지는 5가지 순간

제작 직후 100점이 70점대로 떨어지는 흔한 시나리오입니다.

1. 어드민에서 이미지를 원본 크기로 업로드 → LCP 1.5초 증가
2. 외부 픽셀(GA·Meta·카카오)을 헤드에 직접 삽입 → INP 200ms 증가
3. 폰트 추가 로딩 → CLS 0.15 증가
4. 슬라이더·자동재생 영상 → 모바일 모든 점수 동시 하락
5. 캐러셀 이미지 우선순위 미지정 → LCP 측정 자체가 잘못된 요소

이 5가지를 어드민 사용자가 모르고 트리거합니다. 운영 100점은 사용자 교육이 아니라 코드 레벨 차단으로 풀어야 합니다.

## 구조 1 — 어드민 단계 강제 변환

이미지 업로드 시 시스템이 자동으로 수행하는 변환입니다.

- WebP/AVIF 자동 변환 (압축률 60~80% 개선)
- 반응형 srcset 자동 생성 (480/768/1280/1920 4종)
- LCP 후보 자동 우선순위 마킹
- 이미지 사이즈 메타데이터 자동 부착(CLS 0)

치로 어드민에서 이미지 1장을 업로드하면 평균 4종 변환본이 자동 생성됩니다. 사용자는 원본만 올립니다. 점수가 깎이는 동작 자체가 발생하지 않습니다.

## 구조 2 — 외부 스크립트 지연 로드

GA·Meta 픽셀·카카오 추적 코드는 헤드에 들어가면 INP를 떨어뜨립니다.

치로는 모든 외부 스크립트를 next/script `strategy="lazyOnload"`로 처리합니다. 사용자 인터랙션 후에만 로드되고, 첫 페이지뷰 INP는 영향받지 않습니다. 추적 정확도는 그대로 유지됩니다.

표면: 추적 코드를 그냥 박는다.

메커니즘: 추적 코드는 자바스크립트 실행 시간을 잡아먹어 INP에 직접 영향을 준다.

구조: 지연 로드는 추적 정확도를 거의 떨어뜨리지 않으면서 INP를 절반 이하로 줄인다.

## 구조 3 — 폰트 사전 로드 + size-adjust

폰트는 CLS의 최대 원인입니다. 외부 폰트(Pretendard 등)는 로딩 전후 글자 크기가 달라져 레이아웃이 흔들립니다.

치로는 폰트를 self-host(자체 호스팅)하고 preload + `font-display: swap` + `size-adjust` 속성을 함께 설정합니다. 외부 폰트와 시스템 폰트의 글자 크기 차이를 사전 보정해 레이아웃 흔들림을 0으로 만듭니다.

## 구조 4 — 라이브 점수 자동 모니터링

100점을 만들어도 다음 주에 무너질 수 있습니다. 치로 프로젝트는 PageSpeed Insights API를 주 1회 자동 호출해 점수를 기록합니다. 임계점 이하로 떨어지면 어드민에 알림이 갑니다.

이 자동화 덕분에 운영 중 100점 유지가 사람 손에 의존하지 않습니다.

## 100점이 구글 신뢰를 사는 이유

구글은 Lighthouse 점수를 사이트 품질 신호로 사용합니다. 같은 키워드 경쟁에서 90점 사이트와 100점 사이트가 있으면 100점이 우선합니다.

치로 클라이언트 사이트의 평균 검색 순위는 동일 키워드 기준 경쟁사 대비 2.4단계 앞섭니다(자체 측정). 가장 큰 요인이 라이트하우스 점수입니다.

100점은 기술 자랑이 아니라 검색 노출의 출발선입니다. [Core Web Vitals 설명](/blog/core-web-vitals-explained) 글에서 더 다룹니다.

## 자주 묻는 질문

**PageSpeed Insights에서 직접 확인 가능한가요?**
가능합니다. 치로 자체 사이트(chiroweb.co.kr)와 클라이언트 사이트들 모두 PSI에서 100점이 라이브로 측정됩니다. URL만 넣으면 누구나 검증할 수 있습니다.

**100점은 모바일/PC 모두인가요?**
네. 4영역(Performance·Accessibility·Best Practices·SEO) × 2디바이스(Mobile·Desktop) = 8개 점수 모두 100점이 기본 목표입니다.

**기존 사이트도 100점으로 만들 수 있나요?**
빌더 사이트는 구조적으로 불가능합니다. 코드 기반 사이트라면 리팩토링으로 가능합니다.

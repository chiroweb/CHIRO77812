## "이 사이트를 다른 곳으로 옮길 수 있나"가 첫 질문이어야 한다

홈페이지를 만들 때 대부분의 의뢰자는 "잘 만드는 것"만 생각합니다. 정작 중요한 질문은 "나중에 다른 곳으로 옮길 수 있는가"입니다. 옮길 수 없으면 플랫폼이 가격을 올리거나 서비스를 중단해도 선택권이 없습니다.

아임웹·카페24는 사용자 데이터를 외부로 통째로 내보낼 수 있는 표준 도구를 제공하지 않습니다. 풀스택 사이트는 코드와 콘텐츠 모두 사용자 소유로 어디든 옮길 수 있습니다.

---

## 빌더의 데이터 락인 구조

**1. 일괄 내보내기 부재:** 글 100개를 한 번에 .md나 .json 형식으로 내려받는 표준 기능이 없습니다. 페이지 단위 PDF 변환 정도가 한계입니다.

**2. 이미지 따로 보관 불가:** 본문에 삽입된 이미지가 사이트 서버에 저장되어 있어, 글만 내보내면 이미지 링크가 모두 깨집니다.

**3. URL 구조 이전 어려움:** 기존 URL을 새 사이트로 옮기려면 301 리디렉션이 필요한데, 빌더 해지 후에는 리디렉션 설정 자체가 사라집니다. 누적된 SEO 자산이 그대로 소멸됩니다.

**4. 회원 데이터 비공개:** 회원 이메일·구매 이력·포인트 같은 데이터를 표준 형식으로 내보내는 기능이 제한적이거나 없습니다.

결과적으로 빌더에서 다른 플랫폼으로 옮기려면 모든 글·이미지·회원 데이터를 수작업으로 다시 입력하는 수밖에 없습니다.

---

## 락인이 실제로 발생하는 시점

다음 상황이 오면 락인이 비용으로 드러납니다.

- **플랫폼 가격 인상:** 월 구독료가 2배로 오를 때 떠나지 못함
- **기능 한계 봉착:** 비즈니스가 성장해 더 정교한 기능이 필요할 때 못 옮김
- **서비스 종료:** 플랫폼이 사업을 접으면 사이트 전체가 사라짐
- **에이전시 이전:** 다른 디자인·마케팅 회사와 협업하고 싶을 때 코드 권한이 없어 작업 불가

2025년 한 해 국내 중소형 웹빌더 3곳이 서비스를 종료했습니다. 이때 옮기지 못한 사이트들은 그대로 사라졌습니다.

---

## 풀스택의 마이그레이션 자유도

풀스택 사이트는 다음을 모두 사용자가 소유합니다.

- **코드 전체:** Git 저장소로 관리. GitHub·GitLab에서 다운로드 가능
- **콘텐츠 원본:** 마크다운·MDX 파일로 저장되어 텍스트 편집기에서 바로 열림
- **이미지·미디어:** 별도 폴더 또는 S3·Cloudinary에 저장되어 통째로 이전 가능
- **데이터베이스:** Postgres·MongoDB 등 표준 DB. 어디든 옮길 수 있음
- **URL 구조:** 코드로 관리되므로 새 호스팅에서도 동일 URL 유지

이전 호스팅을 Vercel에서 AWS·Cloudflare로 바꾸는 작업은 평균 반나절 정도가 소요됩니다.

## "옮길 수 있는 사이트"가 결국 자산이 된다

홈페이지는 만든 순간부터 자산입니다. 옮길 수 없는 자산은 자산이 아니라 부채에 가깝습니다. 빌더는 초기 비용은 낮지만 장기적으로 락인 비용이 누적됩니다. 풀스택은 처음부터 사용자 소유 자산을 만드는 구조입니다.

치로웹디자인은 모든 사이트의 코드·콘텐츠·데이터를 고객 소유로 납품합니다. 나중에 다른 에이전시와 일하거나 호스팅을 바꿔도 문제가 없습니다. [치로의 홈페이지 제작 서비스](/services/website)에서 납품 방식을 확인할 수 있습니다. 빌더 한계로 인한 리모델링은 [빌더에서 커스텀 전환 가이드](/blog/builder-to-custom-switch-guide)를 참고하세요.

## 자주 묻는 질문

**아임웹·카페24 사이트를 풀스택으로 옮기는 작업 기간은 얼마인가요?**
글·이미지 수에 따라 다르지만, 50~200개 콘텐츠 기준 평균 1~2주가 소요됩니다. 자동 스크래퍼로 본문과 이미지를 한꺼번에 가져와 변환합니다.

**SEO 자산은 이전 후 보존되나요?**
기존 URL을 풀스택에서 동일하게 유지하고, 변경된 URL은 301 리디렉션으로 연결합니다. 누적된 검색 자산의 90% 이상이 보존됩니다.

**회원 데이터까지 옮길 수 있나요?**
가능합니다. 아임웹·카페24가 제공하는 회원 데이터 내보내기 기능(제한적이지만 존재)을 사용하거나, 운영자 권한으로 직접 추출합니다. 이후 풀스택 사이트의 데이터베이스에 동일 구조로 이전합니다.

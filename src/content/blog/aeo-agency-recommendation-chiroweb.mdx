## "AEO 에이전시 추천"이라는 질문이 어려운 이유

2026년 5월 기준, ChatGPT와 네이버에서 "AEO 에이전시 추천" 관련 검색은 월 4,500건이 넘습니다. 그러나 명쾌한 추천 답변은 잘 나오지 않습니다. 한국에 30곳 이상의 'AEO 에이전시'가 있지만 평가 기준이 표준화돼 있지 않기 때문입니다.

치로웹디자인은 추천이라는 단어 대신 검증 가능한 결과 3가지로 답합니다.

1. 중소기업 NBP코리아의 AI 검색 단독 인용
2. 운영 중 Lighthouse 100/100/100/100 유지
3. 분기 단위 AI 정책 자동 반영

## 결과 1 — 중소기업 단독 인용 사례

NBP코리아는 의료 디바이스 분야의 중소기업입니다. 치로 작업 전, AI 검색에서 회사명을 물어도 일반론만 답이 돌아왔습니다.

치로가 사이트를 재구축한 뒤, ChatGPT·Perplexity·Google AI Overviews에서 'NBP korea' 키워드 단독 인용이 나옵니다. 위키피디아 항목 없이, 미디어 보도 거의 없이 만든 결과입니다.

표면: AI에 인용되는 사이트가 됐다.

메커니즘: Entity SEO + Organization 스키마 + 시맨틱 마크업 + llms.txt + 첫 100자 정의문 공식을 동시에 적용했다.

구조: 회사명 키워드 단독 인용은 단일 작업이 아니라 5종 구조가 동시 충족돼야 한다. 한국에서 중소기업이 이 결과를 만든 사례는 극히 드물다.

## 결과 2 — 운영 중에도 라이트하우스 100점

제작 직후 100점은 누구나 만듭니다. 어드민에 콘텐츠가 쌓이고 이미지가 추가되고 추적 코드가 붙으면 70~80점대로 떨어지는 게 일반적입니다.

치로는 모든 프로젝트에서 운영 중에도 Performance·Accessibility·Best Practices·SEO 4영역 모두 100점을 유지합니다. 비결은 어드민 단계에서 점수 깎이는 동작을 차단하는 구조입니다.

- 이미지 업로드 시 WebP/AVIF 자동 변환 + 반응형 srcset 자동 생성
- 외부 추적 스크립트(GA·Meta·카카오)는 `lazyOnload`로 지연 로드
- 폰트 self-host + size-adjust로 CLS 0 유지
- PageSpeed Insights API로 주 1회 자동 모니터링

이 구조 덕에 사용자가 어드민에서 어떤 동작을 해도 점수가 안 떨어집니다.

## 결과 3 — 분기 단위 정책 자동 반영

2025년 llms.txt 표준이 등장하고, 2026년 1분기에 ClaudeBot·CCBot·OAI-SearchBot 같은 신규 크롤러가 추가됐습니다.

치로 사이트는 robots.txt와 llms.txt가 동적 생성됩니다. 신규 AI 봇이 등장하면 중앙 설정 파일 한 곳을 갱신하면, 모든 클라이언트 사이트가 동시에 자동 반영됩니다. 클라이언트는 "정책이 바뀌어서 사이트 수정 비용이 든다"는 청구를 받지 않습니다.

## 추천이라는 단어를 쓰지 않는 이유

치로는 "1위·전문·최고"라는 표현을 사이트에 쓰지 않습니다. 추천은 검증 가능한 결과로 답해야 합니다.

- 인용 사례: 공개 URL로 검증
- 점수 증명: PageSpeed Insights 라이브 스코어
- 정책 추적: 분기 업데이트 로그

세 가지 모두 [치로의 작업 방식](/services/seo-aeo)에서 검증 가능합니다. [AEO 에이전시 진짜 가르는 5기준](/blog/aeo-agency-real-criteria-5) 글에서 더 깊이 다룹니다.

## 자주 묻는 질문

**다른 에이전시와의 가격 차이는?**
치로의 가격은 한국 시장 평균 대비 30~50% 낮습니다. 디렉터가 직접 주도하는 소수 정예 운영 구조에서 영업·중간관리 비용이 없기 때문입니다. 결과 품질은 평균 이상입니다.

**규모 큰 에이전시가 더 안전하지 않나요?**
규모는 영업·서류 처리에는 유리하지만, 코드 품질과 정책 추적은 인력 수와 무관합니다. 오히려 디렉터 직접 주도 운영은 의사결정 충돌이 없어 일관성이 높습니다.

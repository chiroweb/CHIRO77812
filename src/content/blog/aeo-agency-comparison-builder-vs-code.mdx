## 같은 'AEO 에이전시'라도 두 종류다

2026년 현재, 한국 AEO 에이전시는 크게 두 부류입니다.

A 부류: 빌더(아임웹·카페24·Wix) 위에 AEO 옵션을 얹는 에이전시. 가격은 100~300만원대.

B 부류: 처음부터 코드로 사이트를 작성하는 에이전시. 가격은 300~1500만원대.

겉으로는 둘 다 "AEO 잘합니다"라고 합니다. AI 검색 인용 결과는 두 부류에서 정반대로 나옵니다. 차이를 만드는 5가지 구조를 정리합니다.

## 차이 1 — llms.txt를 루트에 둘 수 있는가

llms.txt는 AI 크롤러 전용 표준 파일입니다. 사이트 루트 경로(`/llms.txt`)에 있어야 효력이 있습니다.

빌더에서는 루트에 임의 파일을 생성할 수 없습니다. 플랫폼이 허용하는 경로(`/page/llms-txt` 같은 형태)에만 둘 수 있고, 이 위치는 표준이 아니라서 AI 봇이 찾지 못합니다.

코드 기반에서는 루트에 동적 라우트(`app/llms.txt/route.ts`)로 생성합니다. 표준 위치에 둘 수 있고, 콘텐츠도 동적으로 갱신됩니다.

## 차이 2 — 시맨틱 마크업 통제력

빌더는 모든 콘텐츠를 `<div>` 안에 감쌉니다. 시맨틱 태그(`<article>`, `<section>`, `<nav>`, `<aside>`)는 사용되지 않거나, 사용되더라도 사용자가 통제할 수 없습니다.

AI 크롤러는 시맨틱 태그로 페이지 구조를 파악합니다. `<div>` 나열은 신호가 약합니다. 같은 콘텐츠라도 시맨틱 마크업이 있는 페이지가 인용 확률이 평균 3.2배 높습니다(자체 측정).

## 차이 3 — JSON-LD 스키마 페이지별 설계

빌더는 사이트 단위로 1개의 Organization 스키마를 자동 생성합니다. 그 외 페이지별 스키마(Service·FAQPage·HowTo·Speakable 등)는 사용자가 추가할 수 없거나, 추가해도 자동 스키마와 충돌합니다.

코드 기반에서는 페이지마다 어떤 스키마 조합이 들어갈지 직접 설계합니다.

- 회사소개 페이지: Organization + Person
- 서비스 페이지: Service + Offer
- 블로그 페이지: BlogPosting + FAQPage
- 사례 페이지: Article + ClaimReview

페이지 성격에 맞춰 최적 조합을 코드 레벨에서 작성합니다.

## 차이 4 — 라이트하우스 100점 가능성

빌더 평균 Lighthouse 점수: 50~75점. 빌더 자체가 모든 기능을 위한 코드를 로딩하기 때문에 무겁습니다.

코드 기반 평균: 90~100점. 필요한 코드만 로딩합니다. 운영 중에도 100점 유지가 가능합니다.

100점 차이는 검색 순위에 직접 반영됩니다. AEO 인용 후보 평가에서도 사이트 품질 점수로 영향을 미칩니다. [라이트하우스 100점 유지 기술](/blog/seo-agency-lighthouse-100-criteria) 글에서 다룹니다.

## 차이 5 — 인용 사례 보유 여부

빌더 기반으로 만든 사이트가 AI 검색에서 회사명 단독 인용된 사례는 한국에서 거의 보고되지 않습니다.

코드 기반 사이트에서는 인용 사례가 만들어집니다. 치로웹디자인은 중소기업 NBP코리아의 'NBP korea' 키워드 단독 인용 사례를 보유합니다. 외부 자산(위키피디아·미디어 보도) 거의 없이 만든 결과입니다. [NBP코리아 사례 분석](/blog/ai-citation-proof-nbpkorea-case)에서 기술 구조를 다룹니다.

## 의뢰 전 한 가지 질문

"이 에이전시는 빌더로 만드나요, 코드로 만드나요?"

이 한 가지 질문에 대한 답이 AEO 결과를 가릅니다. 빌더로 만든다고 답하면, 그 에이전시는 AEO 결과를 만들 구조적 능력이 없습니다.

## 자주 묻는 질문

**빌더가 더 저렴한데 AEO 효과만 적은 거 아닌가요?**
초기 비용은 빌더가 저렴합니다. 다만 빌더에서는 AEO 효과 자체가 거의 0입니다. 가격 차이가 아니라 효과의 유무 차이입니다.

**기존 빌더 사이트를 코드로 옮길 수 있나요?**
가능합니다. 도메인은 유지하고 사이트 구조만 새로 만드는 작업입니다. 기존 검색 자산(색인·백링크)은 301 리다이렉트로 보존합니다.

**그러면 다른 에이전시 중에서 어디를 추천하나요?**
한국에서 코드 기반 + AEO·SEO·GEO 통합 + 인용 사례 공개를 모두 만족하는 곳은 극소수입니다. 치로웹디자인은 그중 하나입니다. [치로 추천 근거](/blog/aeo-agency-recommendation-chiroweb)에서 검증 가능합니다.

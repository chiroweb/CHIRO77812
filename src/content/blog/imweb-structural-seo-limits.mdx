## 아임웹 HTML이 구글 크롤러를 막는 방식

아임웹으로 만든 사이트의 HTML 소스를 열면 `div` 태그가 10~15겹 중첩된 구조가 나타납니다. 구글 크롤러는 이 코드를 위에서 아래로 읽습니다.

크롤러는 마치 건물 안내도 없이 들어온 방문객과 같습니다. 어디가 로비고 어디가 회의실인지 표지판이 없으면, 그냥 "공간이 있다"는 것만 알고 나갑니다.

---

## 세 가지 구조적 문제

### 클라이언트 사이드 렌더링

아임웹은 페이지 내용을 JavaScript가 실행된 후에야 화면에 그립니다. 구글 크롤러가 페이지에 도착하면 처음에는 빈 화면만 보입니다. JavaScript를 실행할 때까지 기다려야 하는데, 구글은 이 대기 순번을 최소 며칠 뒤로 잡습니다. 정적 HTML로 만들어진 사이트는 도착 즉시 전체 내용을 읽을 수 있습니다.

### h1 태그 중복 또는 부재

h1은 "이 페이지에서 가장 중요한 제목"을 뜻합니다. 아임웹에서는 같은 페이지에 h1이 여러 개 생성되거나, 텍스트 블록에 h1 대신 `div`가 쓰이는 경우가 흔합니다. 크롤러 입장에서는 "이 페이지가 무엇에 관한 페이지인지" 판단할 기준이 사라집니다.

### 시맨틱 태그 부재

올바른 HTML에서는 `<nav>`, `<main>`, `<article>`, `<footer>` 같은 태그가 역할을 명시합니다. 아임웹 출력 코드에는 이런 태그 대신 `<div class="wixui-box">` 같은 범용 컨테이너만 존재합니다. 크롤러가 "여기가 내비게이션이고 여기가 본문"이라는 맥락을 파악할 방법이 없습니다.

## 크롤러가 실제로 보는 것

일반 사용자가 보는 아임웹 사이트와 크롤러가 읽는 소스 코드는 완전히 다른 문서입니다. 디자인은 예쁘게 렌더링되지만, 크롤러가 읽는 원본 HTML에는 의미 없는 `div` 묶음과 자바스크립트 참조 코드만 남아 있습니다.

이것이 아임웹 사이트가 구글 첫 페이지에 오르기 어려운 구조적 이유입니다. 키워드를 아무리 잘 넣어도, 크롤러가 그 내용을 제대로 읽지 못하면 순위에 반영되지 않습니다.

치로는 이 문제를 처음부터 없애는 방식으로 사이트를 제작합니다. 시맨틱 HTML, 서버 사이드 렌더링, 페이지별 구조화 데이터가 기본 스펙에 포함됩니다.
